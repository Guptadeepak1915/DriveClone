
# 📁 DriveClone – Cloud File Storage Application (MERN Stack)

DriveClone is a **full-stack cloud file storage web application** inspired by platforms like **Google Drive**.  
This project is built using the **MERN Stack (MongoDB, Express.js, React.js, Node.js)** and allows users to securely **upload, organize, manage, and download files** through a clean and responsive interface.

This project is ideal for **learning real-world full-stack development**, understanding **authentication, file handling**, and building **scalable web applications**.

---

## 🎯 Project Objective

The objective of DriveClone is to:
- Learn how cloud-based file storage systems work
- Implement secure user authentication
- Handle file uploads and folder structures
- Practice MERN stack integration
- Build a portfolio-ready full-stack project

---

## ✨ Features

### 🔐 Authentication & Security
- User Sign Up & Login
- JWT-based authentication
- Protected routes

### 📤 File Management
- Upload files
- Download files
- Delete files [we will add this feature in future]
- Store file metadata in MongoDB



### 🖥️ User Interface
- Clean and simple UI
- Fully responsive design
- Easy navigation

### ⚙️ Backend
- RESTful APIs
- Secure file handling
- MongoDB integration
- Middleware-based architecture

---

## 🛠️ Technology Stack

## 🚀 Installation & Setup

```bash
git clone https://github.com/Guptadeepak1915/DriveClone.git
cd DriveClone
````

### 2️⃣ Backend Setup

Start the backend server:

```bash

```
## 🔐 Environment Variables

| Variable   | Description               |
| ---------- | ------------------------- |
| MONGO_URI  | MongoDB connection string |
| JWT_SECRET | Secret key for JWT        |
| PORT       | Backend server port       |

---

## 🧪 How the Application Works

1. User registers or logs in
2. JWT token is generated and verified
3. User uploads files
4. File metadata is stored in MongoDB
5. Files are stored on the server
6. User can download  files anytime
